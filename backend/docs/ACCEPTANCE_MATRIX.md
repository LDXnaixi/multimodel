# 甲方需求满足度矩阵

## 结论

当前版本按“演示覆盖 + MySQL 持久化 + 本地模型进程调用”补全，已从原通信骨架升级为可运行演示系统。硬件、自主可控第三方报告、驻场、培训视频等合同类交付物不属于代码自动生成范围，需项目交付阶段单独提供。

## 软件能力

| 需求域 | 覆盖状态 | 实现说明 |
| --- | --- | --- |
| B/S 架构 | 满足 | Vue3 前端 + Spring Boot 后端，REST/WebSocket 通信 |
| 微服务边界 | 部分满足 | 以包边界模拟 task/model/data/monitor/report/user/environment 服务，可后续拆分 |
| 模型接入 | 演示满足 | 默认种子模型覆盖 YOLOv7/8、DBNet、CRNN、MobileNet、ResNet50、BERT/LSTM/GRU/Transformer/RoBERTa、DeepSpeech/Whisper、Qwen-VL/Yi-VL/VisualGLM、自研模型 |
| 本地模型调用 | 演示满足 | `LocalModelProcessRunner` 通过 Python 进程 JSON 输入输出；不可用时返回确定性备用结果 |
| 模型生命周期 | 演示满足 | 支持注册、上线、下线、回滚、运行日志 |
| 测试数据管理 | 演示满足 | 数据源、数据集、样本、处理、增强、融合、场景生成均持久化 |
| 样本检索 | 演示满足 | 支持关键词、事件类型、时间范围、标签、数据类型 |
| 任务流程 | 演示满足 | 支持节点依赖、条件表达式、失败策略、校验、仿真、模板、重跑 |
| 实时反馈 | 满足 | WebSocket 推送任务进度、模型结果、流水线结果、监控告警 |
| 资源监控 | 演示满足 | 1 秒采集 CPU/内存/磁盘/网络，GPU 通过本地命令适配 |
| 报告导出 | 演示满足 | 支持 JSON/XML/CSV/PDF |
| 用户登录统计 | 满足 | 记录用户名、IP、时间、模块、访问动作并持久化 |
| 环境一致性 | 演示满足 | 提供环境快照、依赖说明、冲突检查、建议项 |
